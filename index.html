【Hexo】Hexo+github搭建github博客

Posted on 2020-01-22 Edited on 2020-01-23

1.创建个人博客

1.1.在gitHub上new一个repository

1.2.git clone将创建的repository到本地

2.配置环境

2.1.安装Node.js

  • 官网下载
  • 配置环境变量：
    系统变量NODE_PATH，即压缩后的路径，如，D:\NodeJS。
    系统变量PATH，新增，%NODE_PATH%。
  • 检测node.js和npm是否安装成功：
    输入node -v和npm -v。

2.2.安装Hexo

  • 下载
    自定义路径，如，D:\hexo 中打开git bash，输入npm install -g hexo-cli。
    注：cmd中没有成功。
  • 配置环境变量：
    系统变量PATH，新增，如，D:\hexo\node_modules\.bin。
  • 检测hexo是否安装成功：
    输入hexo -v。

3.搭建本地博客

3.1.初始化博客框架

在E:\cymbym.github.io目录下，输入hexo init，会自动生成本地文件。

3.2.基础设置

  • 设置站点名、作者昵称和站点描述等
    打开根目录下的_config.yml修改即可。
  • NexT主题的安装
    根目录下输入git clone https://github.com/theme-next/hexo-theme-next themes/next即可。
  • NexT主题的安装
    根目录下的_config.yml,查找theme字段，将字段改为theme: next。更多可参考。

3.3.将网站部署到个人的GitHub Pages上

  • 安装hexo-deployer-git
    项目根目录下输入npm install hexo-deployer-git --save
  • 修改根目录_config.yml文档中deploy的配置
    文档末输入 :
  • 将本地的静态网页部署到服务器上
    hexo deploy/hexo d

3.4.修改与发布

  • 清空缓存
    hexo clean
  • 生成静态文件
    hexo generate/hexo g
  • 启动服务器，本地预览博客效果
    hexo server/hexo s
    在 http://localhost:4000/ 中访问。

4.写博文

4.1.原始方法

  • 创建新博文
    hexo new <title>
    后，source/_posts目录下会生成相应的.md文件，接下来我们可以编辑该文件。
  • 关于在博文中插入图片
    可以把图片统一放置在source/images目录下，然后在使用时用下述方式引用。 admin:
    deployCommand: hexo-publish.bat hexo g -d
    ```
  • 关于在博文中插入图片
    Hexo Admin可以直接复制图片粘贴，然后自动下载到source/images目录并重命名。但在Windows中粘贴后会出现裂图。这时就需要手动把括号中的前后两个斜杠去掉，就能正常显示，如：

    编辑完毕后，就可以点击Deploy。
